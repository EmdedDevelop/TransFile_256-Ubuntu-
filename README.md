# TransFile256: Консольное многопоточное приложение потоковой компрессии и межпоточного обмена данными (Linux-версия)

Многопоточная программная система на C++, реализующая классический паттерн **Producer-Consumer** (Производитель-Потребитель) в рамках единого адресного пространства под ОС Linux. Приложение эмулирует передачу данных через аппаратное ограничение физического уровня с использованием фиксированной **потокобезопасной очереди** размером **256 байт**, с поддержкой сквозной потоковой компрессии и декомпрессии данных «на лету».

## Ключевая архитектура и технологии
*   **Парадигма:** Многопоточность (POSIX Threads / `std::thread`).
*   **Синхронизация потоков:** Потокобезопасная очередь обмена (FIFO), синхронизированная с помощью примитивов `std::mutex` и `std::condition_variable` с использованием лямбда-предикатов для исключения состояний гонки (Race Conditions) и ложных пробуждений (*Spurious Wakeups*).
*   **Асинхронная конфигурация:** Передача параметров расчета блоков между потоками через примитивы `std::promise` и `std::future`.
*   **Алгоритмы сжатия:** Интеграция библиотеки **zlib** (потоковый многопоточный режим `deflate` / `inflate`).
*   **Устойчивость (Robustness):** Корректная обработка критических граничных условий (*Edge Cases*), включая побайтовую обработку пустых файлов (0 байт) без падения системы и генерации исключений.

## Встроенная телеметрия и профилирование (Telemetry)
Система снабжена модулем `Measure` для сквозного профилирования производительности «на лету» без использования внешних тяжелых профайлеров. В конце работы функция `print_telemetry()` выгружает в консоль следующие метрики:
*   `[MEMORY]` **High-Water Mark:** Пиковый размер очереди (контроль переполнения ОЗУ).
*   `[THREAD]` **Статистика планировщика:** Количество защитных блокировок Потребителя (минимизация нагрузки на CPU).
*   `[TIME]` **Высокоточный бенчмарк:** Время обработки через `std::chrono::steady_clock` с точностью до миллисекунд.
*   `[PERF]` **Пропускная способность (Throughput):** Чистая скорость сквозной обработки потока в МБ/сек.
*   `[VERIFY]` **Побайтовый верификатор:** Контроль бинарного совпадения файлов бит-в-бит.

## Системные требования
*   **Операционная система:** Ubuntu 20.04 LTS / 22.04 LTS (или любой современный Linux-дистрибутив).
*   **Компилятор:** GCC / Clang с поддержкой стандарта **C++11** и выше.
*   **Зависимости:** Установленный пакет библиотеки компрессии `zlib1g-dev`.

## Компиляция и сборка в терминале Ubuntu
1. Установите необходимые зависимости для сборки и линковки библиотеки `zlib`:
   ```bash
   sudo apt update && sudo apt install build-essential zlib1g-dev
   ```
2. Скомпилируйте проект с флагом максимальной оптимизации компилятора по скорости (`-O3`):
   ```bash
   g++ -O3 main.cpp -o TF_256 -lz -lpthread
   ```
   *(Флаг `-lz` осуществляет динамическую линковку с zlib, а `-lpthread` подключает поддержку POSIX-потоков).*

## Инструкция по запуску
Запуск исполняемого файла осуществляется из консоли Linux с передачей трех позиционных аргументов:

```bash
./TF_256 [source_file] [output_file] [compression_mode]
```

### Спецификация аргументов:
*   `[source_file]` — Путь к исходному файлу для передачи.
*   `[output_file]` — Путь для сохранения результирующего файла.
*   `[compression_mode]` — Режим обработки данных:
    *   `0` — Без сжатия (прямая фрагментация файла по 256 байт).
    *   `1` — Активация многопоточной компрессии `zlib`.
